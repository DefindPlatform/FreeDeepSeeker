# Full QA audit — 2026-06-22

## Verdict

The tested build is operational across the proxy, compatibility APIs, CLI, coding agent, Agent Studio, Docker sandbox, package build, and authenticated DeepSeek transport. It is not rated “perfect”: the repository-wide line coverage is below a serious release target, several provider/model variants are not exercised live, and the Chrome extension still lacks an automated real-browser export test.

Strict score: **8.2/10**. No known blocker remains for local use, but the remaining risks below should block a claim of production-grade completeness.

## CLI and Studio controls

- Client: `--url`, `--model`, `--system`, `--session`, `--stream`, `--no-stream`, `--list-models`, `--no-auto-start` and help parsing are covered; live stream/non-stream both returned exact markers.
- Agent: workspace, model, permission mode, undo, init, home-folder protection, project map/JSON, new/private session, dry-run, report, step/tool/time budgets and URL parsing are covered.
- Studio: project/model/permission menus, file filter/viewer, task start/cancel, context reset, output expand, task collapse, memory removal, Git confirmation flow, desktop/mobile layout and console health are covered by live browser plus mocked E2E.
- Browser console: zero relevant warnings/errors in desktop and mobile checks.
- Responsive layout: no horizontal document overflow at 390 px.

## Remaining release risks

1. Raise coverage to at least 70% lines overall and at least 60% for each security-critical module; current entry-point coverage is too low.
2. Add authenticated scheduled tests for reasoner, search and alias models without burning quota on every commit.
3. Automate Chrome-extension export/import against a disposable browser profile and mocked provider domain.
4. Add a package-content snapshot test rather than relying only on the no-glob hygiene rule.
5. Verify the final pushed commit on every Node/OS CI matrix job before calling the release green.
