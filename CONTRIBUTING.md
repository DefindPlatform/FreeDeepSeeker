# CONTRIBUTING

Спасибо за интерес к FreeDeepseekAPI! Этот документ поможет подготовить изменения, PR и баг-репорты.

## Быстрый старт

1. Форкните репозиторий.
2. Сделайте ветку для своей задачи, например `feature/docs-update` или `fix/auth-handling`.
3. Установите зависимости:

```bash
npm ci
```

4. Запустите локальные проверки до создания PR:

```bash
npm run check
```

Отдельно проверить чистоту Git-индекса можно командой `npm run repo:check`. Она отклоняет generated-файлы, локальные сессии, секреты, IDE-кэши и другие артефакты, случайно попавшие под отслеживание.

Если вы меняли UI, предварительно соберите студию:

```bash
npm run studio:build
```

## Стиль PR

- Описывайте проблему и решение в заголовке PR.
- Добавляйте краткое описание в тело PR.
- Указывайте, какие тесты запускали.
- Не включайте приватные файлы (`deepseek-auth.json`, `.env`, cookies и т.п.).

## Тесты

Запускайте быстрые тесты во время разработки и полный gate перед PR:

```bash
npm test
npm run lint
npm run coverage
npm run check
```

Для live smoke-тестов используйте:

```bash
npm run test:live
```

## Документация

- Обновляйте `README.md` и/или файлы в `docs/` при изменении API или поведения.
- Добавляйте примеры запросов и описание новых переменных окружения.

## Безопасность и приватные данные

- Не коммитьте `deepseek-auth.json`.
- Не коммитьте `.env`, `.env.local`, cookies или креды.
- Убедитесь, что в PR нет секретных значений.

## Принятие изменений

PR принимаются после прохождения тестов и ревью. Для UI-изменений требуется сборка `dashboard/`.

Если нужна помощь, оставьте комментарий в issue или PR.
